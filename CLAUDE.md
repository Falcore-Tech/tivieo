# Project: Tivieo

## Overview
- **Type**: Next.js website (Tella.tv-style screen + webcam recorder)
- **Stack**: Next.js 16 (App Router), React 19, Tailwind v4, TypeScript, Supabase
- **Package Manager**: bun (run the shadcn CLI via `npx` only; see Learnings)
- **Started**: 2026-06-03

## Architecture Decisions
- **Storage + DB**: **Cloudflare R2** holds video + thumbnail files (two buckets: private `tivieo-videos` served by presigned GET, public `tivieo-thumbnails` served by its `r2.dev` URL). Supabase keeps Postgres (`recordings` table maps slug → R2 key) + Auth + the transcribe edge function — **not** storage. R2 access layer in `lib/r2/index.ts` (`@aws-sdk/client-s3`). `storage_path`/`thumbnail_path` hold bare R2 keys. Full setup in `docs/r2-storage.md`.
- **Recording**: client-side. Screen (`getDisplayMedia`) + webcam (`getUserMedia`) are composited into one video track on a `<canvas>` (PiP webcam bubble), audio is mixed via `AudioContext`, and the combined stream is recorded with the **native `MediaRecorder`** (chunk accumulation via `ondataavailable`).
- **Recording quality**: fixed **1080p** canvas (`OUTPUT_WIDTH/HEIGHT = 1920×1080` in `use-canvas-compositor.ts`), **30fps**, **VP9** codec (falls back VP8 → default), **explicit bitrate** `videoBitsPerSecond = 8 Mbps` + `audioBitsPerSecond = 128 kbps` in `use-recorder.ts` (~61 MB/min) for crisp screen text + webcam. R2 has no per-file cap, so quality is the only constraint; lower the bitrate constant to trade quality for size.
- **Player**: native styled `<video controls>` (`app/v/[slug]/_components/video-player.tsx`). Reliable for browser-recorded webm.
- **Upload**: browser PUTs the webm directly to R2 via a **presigned PUT URL** (minted by the `createUploadTarget` server action) using **axios** with `onUploadProgress`; the small poster JPEG is uploaded server-side in `createRecording` (`putThumbnail`). Playback/download use presigned GET URLs; thumbnails use the public `r2.dev` URL. The auto poster is the **first** frame: `recorder-studio.tsx` snapshots the canvas on the first `requestAnimationFrame` after `recorder.start` (stored in `posterRef`) rather than at stop time. Backfill existing rows with `bun --env-file=.env.local scripts/backfill-first-frame-thumbnails.ts` (ffmpeg pulls the first frame from a presigned video URL → public thumbnails bucket; `--dry` to preview).
- **Links**: title-derived slug — `slugify(title)-<nanoid(6)>`. Slug edits preserve old links via the `recording_aliases` table (308 redirect on the watch page).
- **Transcription**: async via a Supabase **Database Webhook** (INSERT on `recordings`) → `transcribe` edge function (`supabase/functions/transcribe/`) → **Deepgram** `/v1/listen` (`nova-3`, `utterances`, `detect_language`, `keyterm=Faez`, `topics=true`). The function presigns an R2 GET URL with **aws4fetch** (R2 creds are Supabase function secrets) and hands it to Deepgram as a remote URL. `keyterm=Faez` (nova-3 keyterm prompting) boosts the owner's name so it transcribes correctly at the source. Stores `transcript_text`/`_segments`/`_summary`/`_topics` and a `transcript_status` machine (`none→pending→processing→ready/error`). Segments keep Deepgram per-word timings (`words[]`) for caption splitting. **Summary** is no longer Deepgram's extractive `summarize`; it's a **first-person video description generated by OpenAI `gpt-5.4-mini`** (`summarizeTranscript()`) from the transcript, narrated as the owner ("In this video, I walk through…"), styled by a sample in `SUMMARY_SYSTEM_PROMPT`. Needs the `OPENAI_API_KEY` function secret; null on missing key/empty transcript/failure. Deepgram topics are **English-only**; the function retries transcript-only on failure so transcription never breaks. Watch page renders a same-origin WebVTT `<track>` (`/v/[slug]/captions`), an AI summary + topic chips, and a searchable transcript (clickable timestamps seek the shared `<video>` ref via `VideoProvider`). **Captions are built with Deepgram's own `@deepgram/captions` `webvtt()` at `lineLength = 10`**, so each cue is ≤10 words split on real word timestamps (legacy segments without `words[]` fall back to interpolated text splitting — re-transcribe for exact timing). Setup/secrets/webhook steps in `docs/transcription.md`.
- See `/docs` for full detail.

## Preferences & Rules
- Use **bun** for package management, **axios** for client HTTP, **shadcn + CVA** for primitives.
- Colors/spacing only via design tokens in `app/globals.css` — never hardcode hex/oklch in components.
- Locality of behavior: per-route `_components`/`_hooks`/`_lib`/`_actions.ts`. Only truly shared code in root `lib/` and `components/`.
- **Page width:** use the `max-w-page` token (`--container-page: 1760px` in `globals.css`) — never hardcode `max-w-[1760px]`. The `SiteHeader` and content-heavy shells (`/v`, library, landing features) use `max-w-page` + `px-4 sm:px-6 lg:px-8`; single-column forms stay narrow inside the wide header (login `max-w-sm`, edit `max-w-3xl`, record `max-w-5xl`). See `docs/design-system.md` → Layout width.
- No decorative ambient glow. Componentize aggressively — keep pages short.
- Do not run `next build` unless explicitly asked. Verify via `bun dev`.

## Patterns & Conventions
- Route folders own their feature code (e.g. `app/record/_hooks/use-recorder.ts`).
- Supabase clients live in `lib/supabase/{client,server,admin,proxy}.ts`. The admin (service-role) client bypasses RLS and is server-only; only use it after an ownership/visibility check (signed URLs, public thumbnail URLs).
- Server Actions in `_actions.ts` per route; mutations revalidate affected paths.

## Learnings & Corrections
- Prefer maintained libraries over from-scratch builds; hand-write only the small glue no library covers (the PiP canvas compositor). For this project the recording/playback primitives ended up being the **native** `MediaRecorder` + `<video>`, after RecordRTC (0-byte recordings) and `@vidstack/react` (stalled on tokenized/`Infinity`-duration webm) both failed for our recorded-webm case.
- ❌ `bunx shadcn` crashes on bun 1.1.12 ("Export named 'aborted' not found in module 'util'") → ✅ run shadcn via `npx shadcn@latest`; it still installs deps with bun (detects `bun.lockb`). `init -b <x>` selects the primitive library (`radix`|`base`), NOT the base color.
- ❌ Next.js 16 deprecates `middleware.ts` → ✅ use root `proxy.ts` exporting a function named `proxy`. Never keep both files — Next errors if both exist.
- Lint (react-hooks): don't assign `ref.current` during render (init refs in an effect); don't call `setState` synchronously in an effect (derive the value instead).
- ❌ Side effect inside a `setState` updater → React StrictMode double-invokes updaters in dev, so it runs twice (leaked timers, duplicate recorder instances). ✅ Run side effects outside updaters; make timer start idempotent (clear before re-creating).
- ❌ Resizing the `<canvas>` after `captureStream()` ends the track on Firefox (empty recording) → ✅ use a constant `OUTPUT_WIDTH/HEIGHT` canvas that is never resized.
- ❌ Backgrounded tabs pause `requestAnimationFrame` and clamp `setInterval` to ~1fps, so the captured canvas froze while screen-sharing another window → ✅ compositor paints via rAF when focused and via an inline Blob-URL **Web Worker timer** (~30fps, not throttled) as a fallback.
- ❌ Gating the Web Worker fallback on `document.hidden` only covered tab-switch/minimize. Switching to **another application window** that occludes the browser keeps `visibilityState="visible"` (so `document.hidden` is false) yet Chrome still throttles/pauses rAF — neither driver painted, so the recording froze → ✅ both drivers share one `lastDraw` timestamp; rAF paints when it can, the worker paints only when rAF has gone stale (`now - lastDraw >= targetInterval`). No reliance on `document.hidden`, so occlusion is covered too. (`use-canvas-compositor.ts`)
- Caveat: MediaRecorder webm reports `duration = Infinity` (no duration metadata), so a native seek bar is imperfect until buffered. Patch duration at record time with `fix-webm-duration` before upload.
- ❌ R2 presigned PUT returns 403 `SignatureDoesNotMatch` when the browser's PUT `Content-Type` differs from the type signed into the URL → ✅ sign `video/webm` and send the exact same `Content-Type: video/webm` header (don't let the codec-suffixed recorder mime leak into the PUT). (`lib/r2/index.ts`, `app/record/_lib/upload.ts`)
- R2's `r2.dev` public URL makes a **whole bucket** public, so videos (private) and thumbnails (public) must be **separate buckets** — can't mix public/private in one.

## Dependencies & Tooling
- Recording: native `MediaRecorder` (`app/record/_hooks/use-recorder.ts`). `recordrtc` + `@types/recordrtc` and `@vidstack/react` + `vidstack` remain in `package.json` but are unused (safe to remove).
- Storage: `@aws-sdk/client-s3` + `@aws-sdk/s3-request-presigner` — R2 (S3-compatible) presigned URLs + deletes (`lib/r2/index.ts`). Edge function presigns via `aws4fetch` (esm.sh). `@supabase/supabase-js`, `@supabase/ssr` — auth/DB.
- `next-themes` — dark-mode toggle. `nanoid` — slug suffix. `axios` — HTTP (incl. presigned-PUT upload with progress). `@deepgram/captions` — converts stored segments to WebVTT (`webvtt(response, lineLength)`) in the captions route. `class-variance-authority`, `clsx`, `tailwind-merge`, `lucide-react` — UI.
